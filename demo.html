⚠️ 内存泄漏模拟

点击下方按钮模拟不同类型的内存泄漏场景。打开浏览器开发者工具（F12）查看泄漏警告：

1. 全局变量引用泄漏

对象被全局变量引用，GC 无法回收

2. 事件监听器泄漏

DOM 元素被移除但事件监听器未清理

3. 定时器泄漏

setInterval 未清除，回调函数持有对象引用

4. 闭包泄漏

内部函数持有外部变量引用

5. 缓存泄漏

Map/Set 缓存未清理

点击上方按钮模拟各种内存泄漏场景

💡 使用示例

在实际项目中使用 FinalMem：

// 初始化配置 FinalMem.init({ enabled: true, threshold: 5000, // 超过5秒未回收视为泄漏 interval: 2000 // 每2秒检测一次 }); // 追踪对象 const obj = { data: 'test' }; const id = FinalMem.track(obj, { type: 'object', name: 'myObject' }); // 标记对象应该被销毁 FinalMem.destroy(id); // 或者直接取消追踪 FinalMem.untrack(id); // 清除所有追踪 FinalMem.clearAll();
