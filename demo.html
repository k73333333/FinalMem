⚠️ 内存泄漏模拟

点击下方按钮模拟不同类型的内存泄漏场景。打开浏览器开发者工具（F12）查看泄漏警告：

1. 全局变量引用泄漏

对象被全局变量引用，GC 无法回收

2. 事件监听器泄漏

DOM 元素被移除但事件监听器未清理

3. 定时器泄漏

setInterval 未清除，回调函数持有对象引用

4. 闭包泄漏

内部函数持有外部变量引用

5. 缓存泄漏

Map/Set 缓存未清理

点击上方按钮模拟各种内存泄漏场景

💡 提示：阈值表示对象标记销毁后多少毫秒仍未被回收视为泄漏；检测间隔表示每隔多少毫秒检查一次。

💡 使用示例

在原生 JavaScript 项目中使用：

// 方式一：自动初始化 FinalMem.init({ enabled: true, threshold: 5000, // 超过5秒未回收视为泄漏 interval: 2000 // 每2秒检测一次 }); // 方式二：简单配置 FinalMem.configure({ threshold: 3000 }); // 追踪对象 const obj = { data: 'test' }; const id = FinalMem.track(obj, { type: 'object', name: 'myObject' }); // 标记对象应该被销毁（对象生命周期结束时调用） FinalMem.destroy(id); // 或者直接取消追踪（立即停止追踪） FinalMem.untrack(id); // 清除所有追踪 FinalMem.clearAll();

在 React 项目中使用：

import { useReactMemGuard, MemGuardProvider } from 'final-mem'; // 方式一：使用 Hook 追踪对象 function MyComponent() { const [data, setData] = useState({ items: [] }); // 追踪敏感对象 useReactMemGuard(data, { type: 'state', name: 'componentData' }); return
...
; } // 方式二：使用高阶组件 const TrackedComponent = withMemGuard(MyComponent, { type: 'component', name: 'TrackedComponent' }); // 方式三：全局 Provider（推荐） function App() { return ( ); }

在 Vue 项目中使用：

import { MemGuardVuePlugin, useVueMemGuard } from 'final-mem'; // 方式一：安装插件（推荐） const app = createApp(App); app.use(MemGuardVuePlugin, { enabled: true, threshold: 5000, trackComponents: true // 自动追踪所有组件 }); // 方式二：在组件中使用 export default { setup() { const { track, destroy } = useVueMemGuard(); const data = reactive({ items: [] }); const id = track(data, { type: 'state', name: 'componentData' }); onUnmounted(() => { destroy(id); }); return { data }; } }; // 方式三：使用指令
